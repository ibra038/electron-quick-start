Silent printing not working in combination with deviceName (v5.0.7+)

To reproduce the bug, follow the steps below.

View issue on GitHub

1. Enter your own printer name below

PRINTER_NAME:
System printers found: none

2. Choose a demo to run

# Code Expected Behavior Actual Behavior Demo
1 webContents.print(); Opens the print preview As expected
2 webContents.print({ silent: true }); Prints silently to default printer As expected
3 webContents.print({ silent: true, printBackground: true }) ; Prints silently with background to default printer As expected
4 webContents.print({ silent: true, deviceName: + PRINTER_NAME + }); Prints silently to the predefined printer Nothing is send to the printer & callback will never be called
5 webContents.print({ silent: true, printBackground: true, deviceName: + PRINTER_NAME + }); Prints silently with background to the predefined printer Nothing is send to the printer & callback will never be called

3. Check your terminal for logs

We are using Node.js , Chromium , and Electron .
